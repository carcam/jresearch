  1. Forschungsbereiche

    Forschungszentren arbeiten grundsätzlich in verschiedenen Bereichen. J!Research erlaubt die Kategorisierung von Informationen in Forschungsbereiche. Es bietet Ansichten für die Auflistung von Forschungsbereichen und eine detaillierte Beschreibung eines einzelnen Forschungsbereiches inklusive zugehörigen Projekten, Publikationen und Dissertationen.

  2. Kooperationen

    J!Research bietet Support für Kooperationen welche mit externen Organisationen oder Institutionen bestehen. Ansichten werden sowohl für die Auflistung von Kooperationen als auch für eine detaillierte Ansicht von einzelnen Kooperationen.

  3. Einrichtungen

    Einrichtungen sind zum Verwalten von Apparaturen und Anlagen welche in Forschungsaktivitäten genutzt werden. J!Research erlaubt die Verwaltung von Einrichtungen um Namen, Beschreibung, ein Foto hinzuzufügen und einem Forschungsbereich zuzuordnen.

  4. Geldgeber/Sponsoren

    Geldgeber/Sponsoren stellen externe Institutionen oder Organisationen dar, die Forschungsprojekte unterstützen. J!Research erlaubt die Verwaltung dieser um diese dann einem Projekt zuzuteilen.

  5. Projekte

    Projekte sind die Haupttätigkeiten in einem Forschungscenter. J!Research kann detaillierte Informationen von Forschungsprojekten veröffentlichen, welche sein können der zugeordnete Forschungsbereich, den Fertigstellungs-Status des Projektes, dazugehörige Mitarbeiter (es können Projektleiter definiert werden), Start und Frist des Projektes, Geldgeber, Finanzierungshöhe, Website, Logo und eine Beschreibung. Der Administrator kann konfigurierbare Listen von Projekten oder detaillierte Ansichten veröffentlichen.

  6. Publikationen

    Publikationen sind die Hauptfunktionalität in J!Research. J!Research bietet ein große Sammlung an Funktionen zur Verwaltung von Publikationen. Die nächsten Unterkapitel erklären dies etwas mehr im Detail.

    1. Zitatformate

      J!Research beinhaltet Unterstützung für die meisten verwendeten Zitatformate in den unterschiedlichsten akademischen Bereichen wie:

      • APA (American Psychological Association's Style)
      • MLA (Modern Language Association's Style)
      • Chicago (15th Chicago Manual of Style)
      • IEEE (Institute of Electrical and Electronics Engineers' Style)
      • CSE (Council of Science Editors' Style)
      • Vancouver citation style

    2. Verwaltung

      J!Research wurde so entwickelt um kompatibel mit den Bibtex Publikationstypen zu sein. Seit der Version 1.1 wurden aber auch andere Formate implementiert. Publikationen können als intern markiert werden, was bedeutet, dass diese Publikationen im Forschungscenter selbst mit Hilfe der Organisation erstellt wurde. J!Research drei Ansichten zur Auflistung:

      • Default publications list

        Zeigt eine Auflistung von Publikationen, organisiert nach Kriterien wie Jahr der Publizierung, Typ der Publikation oder Titel. Publikationen werden nach dem ausgewählten Zitatformat dargestellt.

      • Single publication description

      • Tabular presentation

        Zeigt eine Auflistung im Tabellenformat, welche Filter für Team, Forschungsbereiche, Authoren und Typen enthalten.

      Das Admin-Interface für Publikationen erlaubt das Importieren und Exportieren in folgende Formate:

      • Bibtex
      • RIS
      • MODS (Metadata Object Description Schema)

    3. Zitierung

      Forschungsarbeiten beziehen sich auf Wissen, welches vorher schon mal publiziert wurde. J!Research bietet aus diesem Grund die Zitierung von Publikationen - welche sich in der Datenbank befinden - via dem Automatic Citation Plugin welches im Joomla WYSIWYG editor integriert wird.

      Dieser Balken kann nur im Joomla TinyMCE editor gefunden werden, wenn J!Research erfolgreich installiert wurde. Every button inserts a command in the editor that is automatically replaced by the correct cite text in the configured citation style when right parameters are provided. Der Citekey muss außer bei der Bibliographie immer angegeben werden.

      • cite{citekey} Nicht eingeschobene Zitierung einer Publikation. Die Autoren sind das Gesprächsthema in diesem Satz. Diese Art der Zitierung macht keinen Sinn für mehr als einer Publikation. Einige Zitatformate erlauben keine Zitierung in diesem Stil.
      • citep{citekey1, citekey2, ... } Eingeschobene Zitierung mit einer oder mehr Publikationen. Wenn diese Art gewählt wird, stellen die Autoren nicht das Gesprächsthema im Text dar. Einige Zitatformate erlauben keine Zitierung in diesem Stil.
      • citeyear{citekey1, citekey2, ... } Erlaubt die Zitierung mit dem Jahr der Publikation von einem oder mehreren Einträgen.
      • nocite{citekey1, citekey2, ... } Zitat von einer oder mehreren Publikationen, es wird allerdings nichts eingefügt. Diese Elemente werden nur in der Erstellung von Referenzen berücksichtigt, welche mit Hilfe des Kommandos bibliography erstellt wird.
      • bibliography{} Fügt eine Liste von Referenzen ein, die die zitierten Einträge verwendet, mit dem ausgewählten Zitatformat.

      J!Research bietet auch die Möglichkeit diese Kommandos in anderen Editoren zu verwenden. Dieser Balken wird auf der unteren Seite jedes anderen Editors angezeigt.

  7. Mitarbeiter

    J!Research übernimmt die Verwaltung von Mitarbeitern. Basisinformationen wie Name oder E-Mail werden von Joomla übernommen. Ansichten für die Darstellung einer Liste im Tabellenformat und für die Darstellung in einem Coverflow sind möglich. Authentifizierte Benutzer können ihr Profil bearbeiten. Das Profil welches im Frontend angezeigt werden kann beinhaltet persönliche Informationen als auch die letzten Aktivitäten des Benutzers (Publikationen, Dissertationen und auch Projekte). Es ist wichtig zu wissen, dass Mitarbeiter zu Aktivitäten hinzugefügt werden können ohne in Joomla einen Account zu besitzen. Hierbei müssen nur die Namen eingetippt werden, anstatt einen Benutzer auszuählen.

  8. Teams

    Teams gruppieren die J!Research Mitarbeiter in organisatorische Einheiten. Sie werden genutzt um verschiedene Funktionen dem Team bereitzustellen. Diese Funktionen werden im Kapitel "Autorisierung" näher erklärt.

  9. Dissertationen

    Die Verwaltung von Abschluss-Dissertationen ist ebenfalls ein Hauptaugenmerk, weil Studenten und Leiter oft mit diesen konfrontiert sind. Dissertationen können wie andere Einheiten als Liste und als detaillierte Ansicht einer einzelnen Dissertation dargestellt werden.

  10. Autorisierung

    Autorisierung wird benötigt um verschiedene Funktionalitäten im J!Research Frontend nutzen zu können. Um dies zu gewährleisten wurden ACL's in der Datei acl.php implementiert. Die ACL's wurden fest niedergeschrieben, da Joomla in der Version 1.5.x ACL's nicht dynamisch generieren lässt.

    1. Publikationen

      Der Inhalt der Datei acl.php für die Autorisierung von Publikationen sieht wie folgt aus: 

      //Add publications				
$acl->addACL('com_jresearch', 'add', 'users', 'author', 'publications', 'all');
$acl->addACL('com_jresearch', 'add', 'users', 'editor', 'publications', 'all');
$acl->addACL('com_jresearch', 'add', 'users', 'publisher', 'publications', 'all');
$acl->addACL('com_jresearch', 'add', 'users', 'manager', 'publications', 'all');
$acl->addACL('com_jresearch', 'add', 'users', 'administrator', 'publications', 'all');
$acl->addACL('com_jresearch', 'add', 'users', 'super administrator', 'publications', 'all');

//Edit publications
$acl->addACL('com_jresearch', 'edit', 'users', 'registered', 'publications', 'own');
$acl->addACL('com_jresearch', 'edit', 'users', 'author', 'publications', 'own');
$acl->addACL('com_jresearch', 'edit', 'users', 'editor', 'publications', 'all');
$acl->addACL('com_jresearch', 'edit', 'users', 'publisher', 'publications', 'all');
$acl->addACL('com_jresearch', 'edit', 'users', 'manager', 'publications', 'all');
$acl->addACL('com_jresearch', 'edit', 'users', 'administrator', 'publications', 'all');
$acl->addACL('com_jresearch', 'edit', 'users', 'super administrator', 'publications', 'all');

//Remove publications
$acl->addACL('com_jresearch', 'remove', 'users', 'registered', 'publications', 'own');
$acl->addACL('com_jresearch', 'remove', 'users', 'author', 'publications', 'own');
$acl->addACL('com_jresearch', 'remove', 'users', 'editor', 'publications', 'all');
$acl->addACL('com_jresearch', 'remove', 'users', 'publisher', 'publications', 'all');
$acl->addACL('com_jresearch', 'remove', 'users', 'manager', 'publications', 'all');
$acl->addACL('com_jresearch', 'remove', 'users', 'administrator', 'publications', 'all');
$acl->addACL('com_jresearch', 'remove', 'users', 'super administrator', 'publications', 'all');

      Erklärung:

      • (1) stellt die Aktion dar, die der ACL hinzugefügt wird.
      • (2) stellt die Gruppe von Usern da, wofür die Regel gilt.
      • (3) stellt die Funktionalität dar, hier die Publikationen.
      • (4) definiert, welche Elemente davon betroffen sind; own steht dafür, dass nur Elemente vom Benutzer betroffen sind. all betrifft alle Elemente. Achtung: Im Fall von Publikationen müssen diese als Intern markiert sein, damit diese im Frontend dargestellt werden

      Es sind für Publikationen auch Team-Autorisierungen implementiert. Wenn der Mitarbeiter Mitglied des Teams ist (das Team des Publikationserstellers), kann er diese Publikation editieren.

    2. Dissertationen

      Diese selben ACL Regeln wurden für Dissertationen implementiert, allerdings werden diese in der derzeitigen Version noch nicht überprüft. Diese werden in den nächsten Versionen implementiert.
