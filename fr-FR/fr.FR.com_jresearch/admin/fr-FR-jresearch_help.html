  1. Domaines de recherche

    Les centres de recherche oeuvrent souvent dans plusieurs domaines de la connaissance. J!Research permet de classer les informations concernant les domaines de recherche. L'application peut afficher la liste des domaines de recherche disponibles, la description détaillée d'un domaine de recherche particulier, de même que les projets, publications et thèses d'un domaine.

  2. Collaborations

    J!Research peut gérer la liste des collaborations. Une collaboration est une entrée qui contient des informations à propos d'organisations ou d'institutions externes qui collaborent de n'importe quelle manière avec un centre de recherche. Le système comprend également des fonctions pour afficher les détails concernant une collaboration ou la liste de toutes les collaborations disponibles.

  3. Installations

    Les installations sont utilisées pour entreposer des informations à propos des équipements et appareils disponibles pour réaliser des activités de recherche dans le centre. J!Research permet la gestion des installations ; chaque installation comprend un nom, les domaines de recherche actifs, une photo et une description.

  4. Financiers

    Les financiers sont des institutions ou des organisations externes qui investissent dans des projets de recherche. L'application J!Research permet de gérer les financiers associés aux projets.

  5. Projets

    Les projets sont les activités principales au sein des centres de recherche. J!Research permet la publication d'informations détaillées sur les projets de recherche, incluant le domaine de recherche, l'état de progression, les membres associés et les responsables, les dates de début et de fin, les financiers et le niveau de contribution, la page Web du projet, le logo et la description des projets. L'administrateur peut afficher la liste (paramétrable selon plusieurs critères) ou les descriptions des projets.

  6. Publications

    La gestion des publications fait partie des fonctionnalités clés de J!Research. Les prochaines sous-sections détaillent les fonctions disponibles.

    1. Styles de citation

      J!Research supporte les styles de citation les plus répandus dans les milieux académiques pour afficher la liste des publications:

      • APA (American Psychological Association)
      • MLA (Modern Language Association)
      • Chicago (15th Chicago Manual of Style)
      • IEEE (Institute of Electrical and Electronics Engineers)
      • CSE (Council of Science Editors)
      • Vancouver

    2. Gestion

      J!Research a été initialement conçu pour être compatible avec les publications du type Bibtex. Néanmoins, d'autres formats de publications sont supportés depuis la version 1.1. Les publications peuvent être internes ou non. Les articles internes sont écrits avec la participation de l'organisation et peuvent être affichés comme des travaux effectués au centre de recherche. J!Research offre trois vues différentes pour les publications:

      • Liste des publications par défaut

        Affiche la liste des publications, organisée selon des critères tels que l'année de publication, le type ou le titre. Les publications sont affichées selon le style de citation spécifié.

      • Description d'une publication

      • Présentation tabulaire

        Affiche la liste des publications dans un tableau, incluant des filtres selon l'équipe, le domaine de recherche, l'auteur ou le type.

      Les outils d'administration permettent d'importer et d'exporter des publications dans les formats suivants:

      • Bibtex
      • RIS
      • MODS (Metadata Object Description Schema)

    3. Citation

      Les activités de recherche reposent souvent sur les travaux d'autres chercheurs. Pour cette raison, J!Research permet la citation d'articles enregistrés dans sa base de données à partir d'un plugiciel de citation automatique disponible dans l'éditeur WYSIWYG de Joomla.

      Cette barre peut seulement être trouvée dans l'éditeur TinyMCE de Joomla si J!Research a été installé avec succès. Tous les boutons permettent l'insertion d'une commande dans l'éditeur qui est automatiquement remplacée par la citation, selon le style de citation sélectionné. Sauf pour la commande concernant la bibliographie, les clés de citation des publications doivent être saisies obligatoirement.

      • cite{clé_de_citation} Citer une publication sans utiliser les parenthèses. Dans ce cas, les auteurs sont le sujet de la phrase. Ce type de citation n'est pas approprié pour plus d'une publication. Certains styles de citation ne définissent pas les citations sans parenthèses.
      • citep{clé_de_citation1, clé_de_citation2, ... } Citer une ou plusieurs citations en faisant usage de parenthèses. Pour ce type de citation, les auteurs n'apparaissent pas dans le sujet de la phrase. Certains styles de citation ne définissent pas les citations entre parenthèses.
      • citeyear{clé_de_citation1, clé_de_citation2, ... } Citer l'année de publication d'une ou de plusieurs entrées.
      • nocite{clé_de_citation1, clé_de_citation2, ... } Citer une ou plusieurs publications sans rien insérer. Les articles cités seront tout de même considérés lors de la génération de la liste des références à partir de la commande bibliography.
      • bibliography{} Insérer la liste des références selon les enregistrements cités dans le document et le style de citation configuré.

      De plus, J!Research fournit une interface pour citer des articles et générer des listes de références pour les utilisateurs qui n'utilisent pas l'éditeur TinyMCE. Dans ce cas, les boutons apparaissent en dessous de l'éditeur standard.

  7. Personnel

    J!Research permet la gestion des membres du personnel. Les informations standard telles que le nom et le courriel peuvent être importées depuis la table des utilisateurs Joomla. Il est possible de visualiser la liste des membres du personnel dans un tableau ou à l'aide d'un organigramme. En outre, les utilisateurs authentifiés qui ont été enregistrés comme des membres J!Research peuvent modifier leurs profils directement depuis le site. Les pages des profils comprennent des informations personnelles et un sommaire des dernières activités (publications, thèses et projets). Il est important de mentionner que J!Research permet également d'associer des personnes non membres à des activités. Pour ce faire, il suffit d'entrer manuellement leurs noms au lieu de les choisir dans la liste du personnel.

  8. Équipes

    Une équipe comprend plusieurs membres J!Research qui sont classés dans des organisations. Ces équipent servent pour gérer les autorisations et assigner des permissions sur des opérations d'équipes. Ces opérations sont élaborées davantage dans la section "Autorisation".

  9. Thèses

    La gestion des thèses de doctorat fait partie intégrante de J!Research étant donné qu'il est fréquent que le personnel soit impliqué dans ce type d'activités, que ce soit en tant qu'étudiants ou directeurs. La liste des thèses et d'autres documents peuvent être affichés et détaillés sur le site.

  10. Autorisation

    L'autorisation est requise pour plusieurs opérations de J!Research, surtout celles affectant le contenu du site. Pour gérer ces permissions, des fichiers ACL sont implémentés dans le fichier d'aide J!Research acl.php. Les commandes ACL sont inscrites à même ce fichier, car Joomla ne supporte pas les modifications de manière dynamique.

    1. Publications

      Le contenu du fichier acl.php apparaît ci-dessous. Ces commandes permettent de définir des autorisations concernant les publications. 

      //Ajouter les publications
$acl->addACL('com_jresearch', 'add', 'users', 'author', 'publications', 'all');
$acl->addACL('com_jresearch', 'add', 'users', 'editor', 'publications', 'all');
$acl->addACL('com_jresearch', 'add', 'users', 'publisher', 'publications', 'all');
$acl->addACL('com_jresearch', 'add', 'users', 'manager', 'publications', 'all');
$acl->addACL('com_jresearch', 'add', 'users', 'administrator', 'publications', 'all');
$acl->addACL('com_jresearch', 'add', 'users', 'super administrator', 'publications', 'all');

//Modifier les publications
$acl->addACL('com_jresearch', 'edit', 'users', 'registered', 'publications', 'own');
$acl->addACL('com_jresearch', 'edit', 'users', 'author', 'publications', 'own');
$acl->addACL('com_jresearch', 'edit', 'users', 'editor', 'publications', 'all');
$acl->addACL('com_jresearch', 'edit', 'users', 'publisher', 'publications', 'all');
$acl->addACL('com_jresearch', 'edit', 'users', 'manager', 'publications', 'all');
$acl->addACL('com_jresearch', 'edit', 'users', 'administrator', 'publications', 'all');
$acl->addACL('com_jresearch', 'edit', 'users', 'super administrator', 'publications', 'all');

//Enlever les publications
$acl->addACL('com_jresearch', 'remove', 'users', 'registered', 'publications', 'own');
$acl->addACL('com_jresearch', 'remove', 'users', 'author', 'publications', 'own');
$acl->addACL('com_jresearch', 'remove', 'users', 'editor', 'publications', 'all');
$acl->addACL('com_jresearch', 'remove', 'users', 'publisher', 'publications', 'all');
$acl->addACL('com_jresearch', 'remove', 'users', 'manager', 'publications', 'all');
$acl->addACL('com_jresearch', 'remove', 'users', 'administrator', 'publications', 'all');
$acl->addACL('com_jresearch', 'remove', 'users', 'super administrator', 'publications', 'all');

      où:

      • (1) est l'action permise. Dans notre exemple, il s'agit de l'édition d'un document.
      • (2) est le groupe des utilisateurs Joomla sur lequel la règle est appliquée.
      • (3) est l'entité. L'action est appliquée sur l'édition de publications dans le cas présent.
      • (4) définit le niveau d'accès ; own signifie que l'action peut seulement s'effectuer sur les entités de l'utilisateur. Un utilisateur est le propriétaire d'une entité lorsqu'il est défini en tant qu'auteur. all s'applique sur tous les articles publiés. Dans le cas des publications, les articles doivent être marqués en tant que travaux internes afin d'apparaître sur le site.

      Il existe également des autorisations pour les équipes qui sont valables pour les publications. Si un membre fait partie d'une équipe (soit l'équipe du créateur de la publication), l'utilisateur peut modifier la publication.

    2. Thèses

      Les mêmes règles ACL qui ont été mentionnées dans le document sont valables pour les thèses. Cependant, les commandes ACL ne sont pas prises en compte dans la présente version pour ce type de document. Le support du ACL pour les thèses sera ajouté dans les prochaines versions.
